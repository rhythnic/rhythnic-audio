About the Rhythnic Audio Player

The Rhythnic Audio Player is a light-weight, highly customizable audio player for the web. It utilizes the audio tag, javascript, and css to give you a logo-free music player that you can style with css. It isn't built to enable a flash fallback for broswers that don't support the audio tag. The way I see it, if Google stops supporting IE8, then I can too. If a browser doesn't support the audio tag or javascript, the playlist displays as a list of links to audio files.

This audio player isn't a plugin for jQuery. I made it instead as a javascript component, so we won't have the overhead of jQuery. On the downside, there are bound to be issues that come up, but we'll cross those bridges as we come to Internet Explorer. For now, I've kept the player lightweight in anticipation that support for HTML5 standards will only get better. You can change the styling to your heart's content (or to the limits that are imposed, more on that later). The player also uses an icon font system so that you can change the icon's color with css. Let's get into the player a little bit so you can get it working, if this is what you're looking for.

The HTML

Let's go over the code for the HTML file that will include the audio player. We'll start at the top and work down.

The HEAD
Link to Font Awesome to enable the icons to display. Put rhythnic-audio.css in your css folder and link to it. The file main.css refers to whatever your primary css file is for your page. It's not referring to main.css in the zip file (that's for this here page, dag nabbit). If you copy and paste the rules from rhythnic-audio.css into your primary css file, you can omit the link to rhythnic-audio.css.

The HEAD: CSS, Font Awesome, Viewport
  1. <html lang="en">
  2. <head>
  3.   <meta charset="UTF-8">
  4.   <meta name="viewport" content="width=device-width, initial-scale=1" />
  5.   <title>Your page title</title>
  6.   <link href="//netdna.bootstrapcdn.com/font-awesome/4.0.3/css/font-awesome.css" rel="stylesheet">
  7.   <link href="/css/rhythnic-audio.css" rel="stylesheet">
  8.   <link href="/css/main.css" rel="stylesheet">
  9. </head>

FREEDOM! (Voice of Chris Farely: Did you see that part where Mel Gibson was all like "FREEDOM"? That was awesome!)

Let's look at the player HTML. When I say freedom, what I mean is this: you can change the order of the elements inside of the controls div, and you can include multiple lists of type ul or ol, or both, with headers or images in between the lists and all that jazz. But don't get too carried away or it'll become the css version of the Christmas lights knot in Christmas Vacation. You can alter all the colors and styling via css, but we already talked about that.

This example is for a one song player. There's no difference to creating a playlist player or a one banger. The only difference is in the number of songs you include in the list. If there's only one song, the player will remove the icon for showing/hiding the playlist view. The link elements are very important, not just as a fallback, but that's where the player gets the info for the track titles and the paths to the audio files. The javascript uses the class names to find the elements. This allows you the FREEDOM to swap elements around for your design, but make sure to keep the class names or you'll break the player, (classes: rhythnic-audio, controls, play, seek, time, duration, toggle-playlist-view, playlist). If you use the code exactly as is, you shouldn't have any problems, and if it ain't broke don't fix it.

The rhythnic-audio container
  1. <div class="rhythnic-audio" tabindex="0">
  2.   <div class="controls">
  3.     <i class="play fa fa-play fa-fw" title="Play Pause"></i>
  4.     <div>
  5.       <label>Seek</label>
  6.       <input class="seek" type="range" min="0" max="100" value="0" title="Seek" tabindex="-1">
  7.       <span class="time">0:00</span><span class="duration">/0:00</span>
  8.       <i class="toggle-playlist-view fa fa-bars" title="Show Hide Playlist"></i>
  9.     </div>
  10.   </div>
  11.   <div class="playlist">
  12.     <ul>
  13.       <li><a href="audio/buttons-and-bows.mp3" title="Buttons and Bows">Buttons and Bows</a><li>
  14.     </ul>
  15.   </div>
  16. </div>

The audio player uses Font Awesome, a popular icon font brand, which work by assigning classes to i elements. Any i element with an "fa" class is a Font Awesome icon. I chose Font Awesome because it's free and very user friendly. You're welcome to use another icon font system, but you have to change the class names to match whatever you decide to use. You'll also need to specify the new class names in the options object when you create the player object. This is explained better in the options section below.

The CSS

You can link to rhythnic-audio.css in the head or copy and paste the code into your main css file. At the top of the rhythnic-audio.css file, there's a rule for the rhythnic-audio container. Use that to position and size the player. All the elements in the player are sized with ems, so you can use the font-size rule for the rhythnic-audio container to scale the player up and down. If anyone wants to send me an alternate css theme, I'll include it in the package for the player, so that people can have some fast, easy options for a nice-looking player.

With font-size set to 100%, the audio player is at it's natural size. You can adjust the font-size of .rhythnic-audio to increase or decrease the size of the player.

CSS rule for .rhythnic-audio in your main css file
  1. .rhythnic-audio {
  2.     font-size: 100%;
  3. }

One thing to keep in mind is that when you show/hide the playlist tracks via the show/hide button, the size of the player expands and contracts. In the options, you have the choice of disabling the show/hide feature by locking the playlist view, open or close. There's no skip button, so you might not want to lock it closed, but you can hit the up or down arrows to change tracks. You could also specify a height for the player, the height that it has when opened, assuming that you're not using a border and background color for the player. Else, put it in another div and specify the height for that div.

If your curious about the code used to modify the player to achieve the alternate layout at the top of the page, the code for that is at the bottom of the rhythnic-audio.css file.

The Javascript

Put rhythnic-audio.min.js into your script directory (js). At the bottom of our html file, just inside the body closing tage, we'll put our line to rhythnic-audio.js and write the script to create the player.

Link to rhythnic-audio.js
  1. <script src="js/rhythnic-audio.min.js"></script>

This code will create a player for the div contianer with a class of rhythnic-audio. If you have multiple audio players on the same page, repeat the code for each player and use unique class names.

One song player
  1. <script>
  2.   var player = new RhythnicAudio(document.querySelector(".rhythnic-audio"));
  3.   player.init();
  4. </script>

Audio type support

The player's default option for mime type is mp4, because that seems to be the most widely supported. If you're unfamiliar, different browsers support different types of audio files, so the Rhythnic Audio player allows you to include several audio types, just in case. The default mime type (mp4) is referring to the links to the audio files inside of the player html. In my experience, declaring an mp4 mime type also allows for the playback of mp3 files too, but just to be safe, if you're using files that aren't of the mp4 mime type, then override the default mime type by passing the new mime type as an option when you create the player in javascript.

Since we used mp3 files in our example above, we're going to specify the mpeg mime type here.

Specifying an mpeg mime type
  1. <script>
  2.   var player = new RhythnicAudio(document.querySelector(".rhythnic-audio"),
  3.          { "mime" : "mpeg" });
  4.   player.init();
  5. </script>

To add a backup audio type, pass it in with the options object. The "tracks" key refers to another object that holds the file paths. Inside of the tracks object, create keys for the mime types and use sets of strings for your audio paths as the value to those mime type keys.

In this example, we're assuming that we used an mp4 mime type file in our links, and we're providing mp3 and ogg backup files. In reality you only need mp3 or mp4, because all browsers that support mp4 also support mp3 and vice versa.

Adding backup audio files
  1. <script>
  2.   var player = new RhythnicAudio(document.querySelector(".rhythnic-audio"),
  3.          { "tracks" : { "ogg" : [buttons-and-bows.oga],
  4.                         "mpeg" : [buttons-and-bows.mp3] } });
  5.   player.init();
  6. </script>

The Options

tracks
Backup audio file paths, object of key value pairs where the key is the mime type and the value is a set of strings, paths to the audio files.
mime
Mime type for the audio files in your list of links (inside the playlist container), default is mp4.
autoplay
Start playback when the page loads, true or false, default is false.
hidePlaylist
Playlist closes on page load so that only one song is visible, true or false, default is false.
lockPlaylist
Remove the playlist show/hide button from the player. User will be unable to open/close playlist. True or false, default is false.
playIcon
Specify a icon font class for the play icon. Necessary only if you want to change the icon or icon font system. Default is "fa-play".
pauseIcon
Specify a icon font class for the pause icon. Necessary only if you want to change the icon or icon font system. Default is "fa-pause".

All of the options are optional and can be included in any order. In regard to the last two options, they are needed in case you want to change the icon font system. You should also change the class names in the actual "i" elements in the .rhythnic-audio container, if you decide to change icon font systems. Also, false is the default setting for all true/false options. To lock your playlist in the open position, just set lockPlaylist to true.

Accesssibility

I've done a few things to make the player more accessible, in the design and operation of the player. In the design, I made the play button big, and a border that changes color on focus. The player also has a tabindex of 0, so it can be selected using the tab key. If the player is selected, then the user can operate the player via the keyboard. The space bar will play/pause the current track. The up and down arrows will navigate through the tracks in the playlist, and the left/right arrows rewind and fast-forward respectively.
