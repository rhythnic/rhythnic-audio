About the Rhythnic Audio Player

The Rhythnic Audio Player is a light-weight, highly customizable audio player for the web. It utilizes the audio tag, javascript, and css to give you a logo-free music player that you can style with css. It isn't built to enable a flash fallback for broswers that don't support the audio tag. If a browser doesn't support the audio tag or javascript, the playlist displays as a list of links to audio files.

This audio player isn't a plugin for jQuery. I made it instead as a javascript component, so we won't have the overhead of jQuery. I've kept the player lightweight in anticipation that support for HTML5 standards will only get better. You can change the styling to your heart's content (or to the limits that are imposed, more on that later).

The player uses inline svg for buttons so that they will scale well and you can change the color with css. The icons can even be polychrome. The icon system, called Rhythnic Icon, is something I just started to develop. The icons are defined using javascript objects and drawn dynamically. The code for defining and redering the icons takes up just over 1Kb of space in the minified audio player's script file, so it's a very efficient way of using inline svg icons. At present I've only defined the three icons used in the audio player, but you can view the public Git repository at bitbucket.org/rhythnic/rhythnic-icon. Let's get into the player a little bit so you can get it working, if this is what you're looking for.

The HTML

Let's go over the code for the HTML file that will include the audio player. We'll start at the top and work down.

The HEAD
Link to rhythnic-audio.css or copy and paste it into your css file.

The HEAD: CSS, Viewport
  1. <html lang="en">
  2. <head>
  3.   <meta charset="UTF-8">
  4.   <meta name="viewport" content="width=device-width, initial-scale=1" />
  5.   <title>Your page title</title>
  6.   <link href="/css/rhythnic-audio.css" rel="stylesheet">
  7.   <link href="/css/yourcssfile.css" rel="stylesheet">
  8. </head>

This example is for a one song player. There's no difference between creating a playlist player or a one banger. The only difference is in the number of songs you include in the list. If there's only one song, the player will remove the icon for showing/hiding the playlist view. The link elements are very important, not just as a fallback. That's where the player gets the info for the track titles and the paths to the audio files. The javascript uses the class names to find the elements. This allows you the FREEDOM to swap elements around for your design, but make sure to keep the class names or you'll break the player. The icon shapes will be inserted into the SVG tags by javascript. If you use the code exactly as is, you shouldn't have any problems.

If you don't want to use the default icons, you can prevent the player from drawing the icons by passing "drawIcons": false inside of the options object when you create the player. If you substitute icon fonts or images, for example, you need to include 2 elements inside of a div with the class of play, and give the pause element a class of "remove". The CSS

You can link to rhythnic-audio.css in the head or copy and paste the code into your main css file. At the top of the rhythnic-audio.css file, there's a rule for the rhythnic-audio container. Use that to position and size the player. All the elements in the player are sized with em, so you can use the font-size rule for the rhythnic-audio container to scale the player up and down. If anyone wants to send me an alternate css theme, I'll include it in the package for the player, so that people can have some fast, easy options for a nice-looking player.

With font-size set to 100%, the audio player is at it's natural size. You can adjust the font-size of .rhythnic-audio to increase or decrease the size of the player.

CSS rule for .rhythnic-audio in your main css file
  1. .rhythnic-audio {
  2.     font-size: 100%;
  3. }

One thing to keep in mind is that when you show/hide the playlist tracks via the show/hide button, the size of the player expands and contracts. In the options, you have the choice of disabling the show/hide feature by locking the playlist view, open or close. There's no skip button, so you might not want to lock it closed, but you can hit the up or down arrows to change tracks. You could also specify a height for the player, the height that it has when opened, assuming that you're not using a border and background color for the player. Else, put it in another div and specify the height for that div.

If your curious about the code used to modify the player to achieve the alternate layout at the top of the page, the code for that is at the bottom of the rhythnic-audio.css file.

The Javascript

Put rhythnic-audio.min.js into your script directory (js). At the bottom of our html file, just inside the body closing tage, we'll put our line to rhythnic-audio.min.js and write the script to create the player.

Link to rhythnic-audio.js
  1. <script src="js/rhythnic-audio.min.js"></script>

This code will create a player for the div container with a class of rhythnic-audio. If you have multiple audio players on the same page, repeat the code for each player and use unique ids or class names to differentiate the players.

One song player
  1. <script>
  2.   var player = new RhythnicAudio(document.querySelector(".rhythnic-audio"));
  3.   player.init();
  4. </script>

Audio type support

The player's default option for mime type is mp4, because that seems to be the most widely supported. If you're unfamiliar, different browsers support different types of audio files, so the Rhythnic Audio player allows you to include several audio types, just in case. The default mime type (mp4) is referring to the links to the audio files inside of the player html. In my experience, declaring an mp4 mime type also allows for the playback of mp3 files too, but just to be safe, if you're using files that aren't of the mp4 mime type, then override the default mime type by passing the new mime type as an option when you create the player in javascript.

Since we used mp3 files in our example above, we're going to specify the mpeg mime type here.

Specifying an mpeg mime type for html links
  1. <script>
  2.   var player = new RhythnicAudio(document.querySelector(".rhythnic-audio"));
  3.   player.init( {"mime" : "mpeg"} );
  4. </script>

To add a backup audio type, in case a browser doesn't support your main audio type, pass it in with the options object. The "tracks" key refers to another object that holds the file paths. Inside of the tracks object, create keys for the mime types and use sets of strings for your audio paths as the value to those mime type keys. The player uses your mime keys to set the mime type for playback, so make sure your keys specify the correct mime term.

In this example, we're assuming that we used an mp4 mime type file in our links, and we're providing mp3 and ogg backup files. In reality you only need mp3 or mp4, because all browsers that support mp4 also support mp3 and vice versa.

Adding backup audio files
  1. <script>
  2.   var tracks = { "ogg" : ["buttons-and-bows.oga", "One-headlight.oga"],
  3.                  "mpeg" : ["buttons-and-bows.mp3", "One-headlight.mp3"] };
  4.   var player = new RhythnicAudio(document.querySelector(".rhythnic-audio"));
  5.   player.init( {"tracks" : tracks} );
  6. </script>

The Options

Default options
  1. {
  2.   "mime" " "mp4",
  3.   "autoplay" : false,
  4.   "preload" : "metadata",
  5.   "hidePlaylist" : false,
  6.   "lockPlaylist" : false,
  7.   "drawIcons": true
  8.   "tracks" : {},
  9. }
mime
Mime type for the audio files in your list of links (inside the playlist container)
autoplay
Start playback when the page loads.
preload
Sets the preload attribute for the audio element.
hidePlaylist
Playlist closes on page load so that only one song is visible.
lockPlaylist
Remove the playlist show/hide button from the player. User will be unable to open/close playlist.
drawIcons
Should the player draw the default icons. Set to false if you are using other icons.
tracks
Backup audio file paths, object of key value pairs where the key is the mime type and the value is a set of strings, paths to the audio files.

All of the options are optional and can be included in any order.

Accesssibility

I've done a few things to make the player more accessible, in the design and operation of the player. In the design, I made the play button big, and a border that changes color on focus. The player also has a tabindex of 0, so it can be selected using the tab key. If the player is selected, then the user can operate the player via the keyboard. The space bar will play/pause the current track. The up and down arrows will navigate through the tracks in the playlist, and the left/right arrows rewind and fast-forward respectively.
