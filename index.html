About the Rhythnic Audio Player

The Rhythnic Audio Player is a a light-weight, highly customizable audio player for the web. It utilizes the audio tag, javascript, and css to give you a logo-free music player that you can style with css. It isn't built to enable a flash fallback for broswers that don't support the audio tag. The way I see it, if Google stops supporting IE8, then I can too. In the case that javascript is disabled, the browsers default audio tag will display.

The HTML

Let's go over the code for the HTML file that will include the audio player. We'll start at the top and work down.

The HEAD
Link to font awesome to enable the icons to display. Put audioplayer.css in your css folder and link to it. The file main.css refers to whatever your primary css file is for your page. It's not referring to the main.css in the zip file. If you copy and paste the rules from audioplayer.css into your primary css file, you can omit the link to audioplayer.css. Add a link to the custom Modernizr file and add the class "no-js" to the HTML tag so that we can control what happens if javascript is disabled.

The HEAD: CSS, Font Awesome, Viewport
  1. <html lang="en" class="no-js">
  2. <head>
  3.   <meta charset="UTF-8">
  4.   <meta name="viewport" content="width=device-width, initial-scale=1" />
  5.   <title>Your page title</title>
  6.   <link href="//netdna.bootstrapcdn.com/font-awesome/4.0.3/css/font-awesome.css" rel="stylesheet">
  7.   <link href="/css/audioplayer.css" rel="stylesheet">
  8.   <link href="/css/main.css" rel="stylesheet">
  9.   <script src=js/modernizr.custom.59041.js></script>
  10. </head>

Next, let's look at the HTML for the audio player. The first thing is the audio tag. The audio tag is here in case the user has javascript disabled. You can choose which file types you want to support inside of the audio tag. I would suggest ogg/oga for Firefox and Opera and either mp4/mp3/aac for other browsers. These references to the files aren't accessed by the Rhythnic Audio Player. They are just there for use by the audio tag as a fallback. You can also put a Flash player fallback inside of the audio tag, in case the browser doesn't support the audio tag. If either javascript or the audio tag are disabled, the Rhythnic Audio Player isn't displayed and only the audio tag is displayed. If either javascript or the audio tag are disabled, the Rhythnic Audio Player isn't displayed and only the audio tag is displayed. If broswer supports the audio tag, you'll see the single track player. If it doesn't support the audio tag, whether or not javascript is enabled, you'll see the Flash player fallback. The players for a single track and playlists look very similar. You don't have to differentiate in the code. It's determined automatically by how many list items you have in you list.

The audio player uses Font Awesome, a popular brand of font icons, which work by assigning classes to i elements. Any i element with an "fa" class is a Font Awesome icon. The other class names assist in the functioning of the plugin. I chose Font Awesome because it's free and very user friendly.

I chose to include the entire Font Awesome collection by linking to it in the head section for a few reasons, even though we're downloading more than is necessary. The first reason is that it keeps the audioplayer package simple. Also, the less I integrate the font icon system, the easier it is for developers to integrate the audioplayer plugin with the other font icons on their site, whether they are using Font Awesome or another font icon system. You're welcome to use another font icon system, but you have to change the class names to match whatever you decide to use. You'll also need to specify the new class names in the options object when you create the jQuery object. This is explained better in the options section below. Beginning developers will also benifit from the simplicity of linking to Font Awesome in the head.

The CSS

You can link to audioplayer.css in the head or copy and paste the code into your main css file. The Rhythnic Audio Player uses jQuery, so at the top of the audioplayer.css file, I've pasted the necessary code for the jQuery slider. In your main css file, make a rule for the audioplayer so that you can control the size and position of the audioplayer. If you want to, you can change the whole look of the audio player by changing the css in audioplayer.css. If I have time down the road, I'll create some more themes for the audio player, or if anyone wants to send me one to post on this tutorial for download, my email address is at the bottom of the page.

With font-size set to 100%, the audio player is at it's natural size. You can adjust the font-size of .audioplayer to increase or decrease the size of the player. To prevent the playlist container from changing height when you open and close the playlist, specify a height (not necessary for single song players).

CSS rule for .audioplayer in your main css file
  1. .audioplayer {
  2.     font-size: 100%;
  3.     height: 12em;
  4. }

The Javascript

Put audioplayer.min.js, modernizr.custom.59041.js, and jquery.ui-slider.js in your script directory (js). The Rhythnic audioplayer depends on jQuery and the other js files, so you have to link to them before linking to the audioplayer.js file. The best place to include scripts, Modernizr excluded, is at the bottom of the body container.

The first 2 lines get jQuery from Google's CDN with jquery.com as a fallback. The 3rd and 4th line import the jquery.ui-slider and audioplayer javascript files. Inside of jQuery.ui-slider.js I've cut and pasted the code for touch-punch, a jQuery plugin that enables touch events for jQuery ui.

Link to jQuery, jquery ui slider, and audioplayer
  1. <script src = "//ajax.googleapis.com/ajax/libs/jquery/1.10.2/jquery.min.js"></script>
  2. <script>window.jQuery || document.write('<script src="//codeorigin.jquery.com/jquery-1.10.2.min.js"><\/script>')</script>
  3. <script src="js/jquery-ui-slider.js"></script>
  4. <script src="js/audioplayer.min.js"></script>

This code will create the audioplayer for all divs that have a class of audioplayer. If you have multiple audio players on the same page, replace the ".audioplayer" query with a class or id that is unique to each player. The Rhythnic Audio Player supports 3 MIME types, mp4, ogg, and mpeg. To support all browsers, you should include an ogg file and either an mp4 or mpeg file for each song in the playlist. Use the MIME type as the keys for the dictionaries in the playlist.

One song player

  2.   (function ($) {
  3.     $('div.audioplayer').audioplayer({

  6.           ogg: "path/to/ogg/file.ogg",
  7.           mpeg: "path/to/mp3/file.mp3"
  8.       }]
  9.     });
  10.   })(jQuery);
  11. </script>

The Options

playlist
This is a list of dictionaries for the paths to your audio files.
autoplay - true/false
Start playback when the page loads
hidePlaylist - true/false
Hide all song titles after the first song. User will need to expand the list via the button.
lockPlaylist - true/false
Remove the playlist show/hide button from the player. User will be unable to hide or show the playlist.
playIcon
Specify a font icon class for the play icon.
pauseIcon
Specify a font icon class for the pause icon.
muteIcon
Specify a font icon class for the muted icon.
unmuteIcon
Specify a font icon class for the unmute.

All of the options, except for the playlist parameter, are optional and can be included in any order. In regard to the last four options, they are needed in case you want to change the font icon system. You should also change the class names in the actual "i" elements in the .audioplayer container, if you decide to change font icon systems. Also, false is the default setting for all true/false options.

To create an alternate look for your playlist as shown at the top of the page, set hidePlaylist to false and lockPlaylist to true. This will lock your playlist song titles to be permanantly visable. You can then give that audio player another class and style the list(s) however you would like.

Rhythnic Audio Player is open source

This plugin is open source and there is a public repository for it on Bitbucket called rhythnic/audio-player. There are a few design choices that I'm considering for the future.

  • Smaller single-song player
  • Pop-up / drop-down vertical sliders for the volume and seek
  • Skip back button for playlist navigation
  • Scrolling song titles
  • Switching from jQuery UI to jQuery Mobile

I will continue to develop it here and there in the future as time allows, but anyone is able to fork the git project and continue development.
