About the Rhythnic Audio Player

The Rhythnic Audio Player is a a light-weight, highly customizable audio player for the web. It utilizes the audio tag, javascript, and css to give you a logo-free music player that you can style with css. It isn't built to enable a flash fallback for broswers that don't support the audio tag. The way I see it, if Google stops supporting IE8, then I can too. If a browser doesn't support the audio tag or javascript, the playlist displays as a list of links to audio files.

This audio player isn't a plugin for jQuery. I made it instead as a javascript component, so we won't have the overhead of jQuery. On the downside, there are bound to be issues the come up, but we'll cross those bridges as we come to Internet Explorer. For now, I've kept the player lightweight in anticipation that support for HTML5 standards will only get better. You can change the styling to your heart's content (or to the limits that are imposed, more on that later). The player also uses font icons so that you can change their color with css. Let's get into the player a little bit so you can get it working, if this is what you're looking for.

The HTML

Let's go over the code for the HTML file that will include the audio player. We'll start at the top and work down.

The HEAD
Link to Font Awesome to enable the icons to display. Put rhythnic-audio.css in your css folder and link to it. The file main.css refers to whatever your primary css file is for your page. It's not referring to the main.css in the zip file (that's for this here page, dag nabbit). If you copy and paste the rules from rhythnic-audio.css into your primary css file, you can omit the link to rhythnic-audio.css.

The HEAD: CSS, Font Awesome, Viewport
  1. <html lang="en">
  2. <head>
  3.   <meta charset="UTF-8">
  4.   <meta name="viewport" content="width=device-width, initial-scale=1" />
  5.   <title>Your page title</title>
  6.   <link href="//netdna.bootstrapcdn.com/font-awesome/4.0.3/css/font-awesome.css" rel="stylesheet">
  7.   <link href="/css/rhythnic-audio.css" rel="stylesheet">
  8.   <link href="/css/main.css" rel="stylesheet">
  9. </head>

FREEDOM! (Voice of the late Chris Farely: Did you see that part where Mel Gibson was all like "FREEDOM"? That was awesome!)

Let's look at the player HTML. When I say freedom, what I mean is this: you can change the order of the elements inside of the controls div, and you can include multiple lists of type ul or ol, or both, with headers or images in between and all that jazz. But don't get too carried away or it'll become the css version of a Christmas lights knot. You can alter all the colors and styling via css, but we already talked about that.

This example is for a one song player. There's no difference to creating a playlist player or a one banger. The only difference is in the number of songs you include in the list. If there's only one song, the player will remove the icon for showing/hiding the playlist view. The link elements are very important, not just as a fallback, but that's where the player gets the info for the track titles and the paths to the audio files. Make sure to keep all the class names the same (with the exception of changing the font icon system, but more on that later). The javascript uses the class names to find the elements. If you use the code exactly as is, you shouldn't have any problems, and if it ain't broke don't fix it.

The rhythnic-audio container
  1. <div class="rhythnic-audio" tabindex="0">
  2.   <div class="controls">
  3.     <i class="play fa fa-play fa-fw" title="Play Pause"></i>
  4.     <div>
  5.       <label>Seek</label>
  6.       <input class="seek" type="range" min="0" max="100" value="0" title="Seek" tabindex="-1">
  7.       <span class="time">0:00</span><span class="duration">/0:00</span>
  8.       <i class="toggle-playlist-view fa fa-bars" title="Show Hide Playlist"></i>
  9.     </div>
  10.   </div>
  11.   <div class="playlist">
  12.     <ul>
  13.       <li><a href="audio/buttons-and-bows.mp3" title="Buttons and Bows">Buttons and Bows</a><li>
  14.     </ul>
  15.   </div>
  16. </div>

The audio player uses Font Awesome, a popular brand of font icons, which work by assigning classes to i elements. Any i element with an "fa" class is a Font Awesome icon. The other class names assist in the functioning of the plugin. I chose Font Awesome because it's free and very user friendly.

I chose to include the entire Font Awesome collection by linking to it in the head section for a few reasons, even though we're downloading more than is necessary. The first reason is that it keeps the rhythnic-audio package simple. Also, the less I integrate the font icon system, the easier it is for developers to integrate the rhythnic-audio plugin with the other font icon systems. You're welcome to use another font icon system, but you have to change the class names to match whatever you decide to use. You'll also need to specify the new class names in the options object when you create the jQuery object. This is explained better in the options section below.

The CSS

You can link to rhythnic-audio.css in the head or copy and paste the code into your main css file. At the top of the rhythnic-audio.css file, there's a rule for the rhythnic-audio container. Use that to position and size the player. All the elements in the player are sized with ems, so you can use the font-size rule for the rhythnic-audio container to scale the player up and down. If anyone wants to send me an alternate css theme, I'll try to include it in the package for the player, so that people can have some fast, easy options for a nice-looking player.

With font-size set to 100%, the audio player is at it's natural size. You can adjust the font-size of .rhythnic-audio to increase or decrease the size of the player.

CSS rule for .rhythnic-audio in your main css file
  1. .rhythnic-audio {
  2.     font-size: 100%;
  3. }

One thing to keep in mind is that when you show/hide the playlist tracks via the show/hide button, the size of the player expands and contracts. In the options, you have the choice of disabling the show/hide feature by locking the playlist view, open or close. There's no skip button, so you might not want to lock it closed, but you can hit the up or down arrows to change tracks. You could also specify a height for the player, the height that it has when opened, assuming that you're not using a border and background color for the player. Else, put it in another div and specify the height for that div.

If your curious about the code used to modify the player to achieve the alternate layout at the top of the page, the code for that is at the bottom of the css file.

The Javascript

Put rhythnic-audio.min.js, modernizr.custom.59041.js, and jquery.ui-slider.js in your script directory (js). The Rhythnic rhythnic-audio depends on jQuery and the other js files, so you have to link to them before linking to the rhythnic-audio.js file. The best place to include scripts, Modernizr excluded, is at the bottom of the body container.

The first 2 lines get jQuery from Google's CDN with jquery.com as a fallback. The 3rd and 4th line import the jquery.ui-slider and rhythnic-audio javascript files. Inside of jQuery.ui-slider.js I've cut and pasted the code for touch-punch, a jQuery plugin that enables touch events for jQuery ui.

Link to jQuery, jquery ui slider, and rhythnic-audio
  1. <script src = "//ajax.googleapis.com/ajax/libs/jquery/1.10.2/jquery.min.js"></script>
  2. <script>window.jQuery || document.write('<script src="//codeorigin.jquery.com/jquery-1.10.2.min.js"><\/script>')</script>
  3. <script src="js/jquery-ui-slider.js"></script>
  4. <script src="js/rhythnic-audio.min.js"></script>

This code will create the rhythnic-audio for all divs that have a class of rhythnic-audio. If you have multiple audio players on the same page, replace the ".rhythnic-audio" query with a class or id that is unique to each player. The Rhythnic Audio Player supports 3 MIME types, mp4, ogg, and mpeg. To support all browsers, you should include an ogg file and either an mp4 or mpeg file for each song in the playlist. Use the MIME type as the keys for the dictionaries in the playlist.

One song player
  1. <script>
  2.   (function ($) {
  3.     $('div.rhythnic-audio').rhythnic-audio({
  4.       playlist: [{
  5.           mp4: "path/to/mp4/file.m4a",
  6.           ogg: "path/to/ogg/file.ogg",
  7.           mpeg: "path/to/mp3/file.mp3"
  8.       }]
  9.     });
  10.   })(jQuery);
  11. </script>

The Options

playlist
This is a list of dictionaries for the paths to your audio files.
autoplay - true/false
Start playback when the page loads
hidePlaylist - true/false
Hide all song titles after the first song. User will need to expand the list via the button.
lockPlaylist - true/false
Remove the playlist show/hide button from the player. User will be unable to hide or show the playlist.
playIcon
Specify a font icon class for the play icon.
pauseIcon
Specify a font icon class for the pause icon.
muteIcon
Specify a font icon class for the muted icon.
unmuteIcon
Specify a font icon class for the unmute.

All of the options, except for the playlist parameter, are optional and can be included in any order. In regard to the last four options, they are needed in case you want to change the font icon system. You should also change the class names in the actual "i" elements in the .rhythnic-audio container, if you decide to change font icon systems. Also, false is the default setting for all true/false options.

To create an alternate look for your playlist as shown at the top of the page, set hidePlaylist to false and lockPlaylist to true. This will lock your playlist song titles to be permanantly visable. You can then give that audio player another class and style the list(s) however you would like.

Here's the javascript for the alternate playlist layout. Here I've set lockPlaylist to true and hidePlaylist is false by default.

Javascript for the playlist with an alternate layout

  4.       playlist:
  5.         [{
  6.           mpeg: "audio/A-Tired-Apartment.mp3",
  7.           ogg: "audio/A-Tired-Apartment.ogg",
  8.         },{
  9.           mpeg: "audio/Ego-Tripping.mp3",
  10.           ogg: "audio/Ego-Tripping.ogg",
  11.         },{
  12.           mpeg: "audio/Lies-Of-Liars.mp3",

  21. </script>

Rhythnic Audio Player is open source

This plugin is open source and there is a public repository for it on Bitbucket called rhythnic/audio-player. There are a few design choices that I'm considering for the future.

  • Smaller single-song player
  • Pop-up / drop-down vertical sliders for the volume and seek
  • Skip back button for playlist navigation
  • Scrolling song titles
  • Switching from jQuery UI to jQuery Mobile

I will continue to develop it here and there in the future as time allows, but anyone is able to fork the git project and continue development.

The Rhythnic Audio Player jQuery Plugin is released under the open source software license. Anyone may download it for free, change it at will, and use it for commercial purposes.
